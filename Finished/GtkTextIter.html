gtk_text_iter_get_slice ()

gchar *
gtk_text_iter_get_slice (const GtkTextIter *start,

Returns the text in the given range. A “slice” is an array of characters encoded in UTF-8 format, including the Unicode “unknown” character 0xFFFC for iterable non-character elements in the buffer, such as images. Because images are encoded in the slice, byte and character offsets in the returned array will correspond to byte offsets in the text buffer. Note that 0xFFFC can occur in normal text as well, so it is not a reliable indicator that a pixbuf or widget is in the buffer.

返回指定区间的文本。“片”是一组用UTF-8格式编译的字符，包括为缓冲区中的可迭代的非字符元素提供的统一码“unknown”字符0xFFFC，例如图像。因为图像能够被编译成片，字节和字符偏移量在一个返回数组中将符合在文本缓冲中的字节偏移量。记住，0xFFFC 也能够出现在正常的文本中，所以这不是一个可靠的指标来表明像素缓冲或者控件是否在缓冲区中。

Returns TRUE if tag is toggled on at exactly this point. If tag is NULL, returns TRUE if any tag is toggled on at this point.

Note that if gtk_text_iter_begins_tag() returns TRUE, it means that iter is at the beginning of the tagged range, and that the character at iter is inside the tagged range. In other words, unlike gtk_text_iter_ends_tag(), if gtk_text_iter_begins_tag() returns TRUE, gtk_text_iter_has_tag() will also return TRUE for the same parameters.

返回 TRUE 如果 tag 恰在这点上是打开的。如果 tag是NULL，返回 TRUE如果在这点上任何标记都是打开的。
